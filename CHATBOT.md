# TechFlare Solutions AI Chatbot — Official Guide

The **TechFlare Solutions AI** assistant appears on every page of the main website (bottom-right). It answers questions about services, products, Innovation Hub, **terms & payment policy**, portal workflow, and contact details.

---

## How it works

1. Visitor opens the chat widget (no login required).
2. The frontend sends the message to `POST /api/ai/chat` on the backend.
3. The backend:
   - Matches keywords against a **knowledge base** (instant answers).
   - If no match and an LLM API key is set, calls **Groq** (preferred) or **OpenAI** with the full **official policy document** in the system prompt.
   - Falls back to knowledge-base answers if the LLM fails.

---

## Official policy document

The chatbot is trained on TechFlare's official document, built from:

| Source | Location |
|--------|----------|
| Terms & Conditions | `backend/src/data/policies.ts` → `termsSections` |
| Payment policy (60/40, refunds, delays) | `paymentPolicy` |
| Portal workflow & milestones | `workflowStages` |
| Review SLA (24 working hours) | `reviewSla` |

The combined text is generated by `buildPolicyDocument()` and injected into `AI_SYSTEM_PROMPT` in `backend/src/lib/ai-knowledge.ts`.

Public copy of terms: **https://techflaresolutionss.vercel.app/terms**

## API keys (Render backend)

Set these on **Render** → `techflaresolutionsback` → **Environment**:

```env
# Recommended — free tier at https://console.groq.com
GROQ_API_KEY=gsk_xxxxxxxx
GROQ_MODEL=llama-3.3-70b-versatile

# Optional fallback
OPENAI_API_KEY=sk-xxxxxxxx
OPENAI_MODEL=gpt-4o-mini
```

| Key | Required? | Notes |
|-----|-----------|-------|
| `GROQ_API_KEY` | Recommended | Fast, cost-effective; used first if set |
| `OPENAI_API_KEY` | Optional | Used only when Groq key is not set |
| Neither | OK | Chatbot still works via keyword knowledge base |

After adding keys, **redeploy** the backend on Render.

---

## Public access (important)

The chatbot **does not require sign-in**. It is rate-limited (30 requests/minute per IP) and origin-checked against `CORS_ORIGINS`.

The **Solutions Center AI proposal** (`POST /api/ai/proposal`) still requires a signed-in account.

## Suggested questions (quick menu)

Default chips in the chat footer:

- What are your terms and payment policy?
- How does project tracking work in the portal?
- How does the Innovation Hub work?
- What services does TechFlare offer?

The menu **stays visible while typing and while the AI is answering**. It hides only after the user sends their first message (can be restored with “Show suggested questions”).

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| “Authentication required” | Redeploy backend — chat route must be public (no `requireVerifiedAuth`) |
| Generic / short answers only | Add `GROQ_API_KEY` or `OPENAI_API_KEY` on Render and redeploy |
| “Connection issue” | Check `NEXT_PUBLIC_API_URL` on Vercel main site points to Render backend |
| “Too many requests” | Wait 1 minute; rate limit is 30/min |
| Menu disappears when typing | Fixed — footer is sticky; menu hides only after first sent message |

## Health check

```bash
curl -X POST https://techflaresolutionsback.onrender.com/api/ai/chat \
  -H "Content-Type: application/json" \
  -H "Origin: https://techflaresolutionss.vercel.app" \
  -H "User-Agent: Mozilla/5.0" \
  -d '{"message":"What services do you offer?","history":[]}'
```

Expected: `200` with `{ "reply": "...", "llm": true|false }`
